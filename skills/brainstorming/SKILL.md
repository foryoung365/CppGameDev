# Brainstorming Into A Full Plan

Use this skill only after the task has been routed to `full-plan`.
It is not the universal first hop. If the task is `micro-plan`, `short-plan`, or `debugging-plan`, skip this skill and use the routed plan shape instead.

## Purpose

Turn a broad request into a clarified design that is ready for `writing-plans`.

## Rules

- Start from the current project context.
- Ask one question at a time.
- If the request is too large for one feature or fix, split it into smaller, independent pieces before going further.
- Propose 2-3 approaches with trade-offs when the design is not obvious.
- Present the design in sections and get agreement before moving on.
- Do not take implementation actions until the design is approved.

## What To Cover

- Goal and success criteria
- Scope boundaries
- Data flow or state impact
- Risks and open questions
- Validation shape for the eventual plan

## Handoff

When the design is clear and approved, hand off to `writing-plans` using the `full-plan` shape.
